## рџљЂ Quick Start

### 1. Clone & set up the environment

```bash
git clone https://github.com/MinoliPerera021126/car-price-prediction-ml-system.git
cd car-price-prediction-ml-system

python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Train the model (optional вЂ” pre-trained artifacts are included)

Open `notebooks/car-price-model.ipynb` in JupyterLab and run all cells.  
The final cell saves the trained model and encoders into the `models/` directory.

### 3. Run the Streamlit app

```bash
streamlit run app/streamlit_app.py
```

The app opens at **http://localhost:8501**.

## рџ§  Model Pipeline

| Stage | Details |
|---|---|
| **Data** | 99 000+ UK used-car listings across 9 brands (Kaggle) |
| **Cleaning** | Remove duplicates, IQR-based outlier removal on price |
| **Feature Engineering** | `car_age`, `mileage_per_year`, `engine_tax_ratio`, `is_premium` |
| **Encoding** | One-hot for `brand`, `transmission`, `fuelType`; target encoding for `model` |
| **Model** | `RandomForestRegressor` with GridSearchCV hyperparameter tuning |
| **Evaluation** | RВІ в‰€ 0.96 on held-out test set |

### Feature Columns (21 total)

```
model, mileage, mpg, engineSize, tax, car_age, mileage_per_year,

fuelType_hybrid, fuelType_petrol, brand_bmw, brand_ford, brand_hyundai,
```

## рџ—ѓпёЏ Dataset

The training data comes from the [100,000 UK Used Car Dataset](https://www.kaggle.com/datasets/adityadesai13/used-car-dataset-ford-and-mercedes) on Kaggle.  
Each CSV contains columns: `model`, `year`, `price`, `transmission`, `mileage`, `fuelType`, `tax`, `mpg`, `engineSize`.

The `data/` directory is **gitignored**. To reproduce training, download the CSVs from Kaggle or use `kagglehub` inside the notebook.
